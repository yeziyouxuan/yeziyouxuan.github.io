一、Hibernate框架

1.hibernate入门
  1. Hibernate:是一个开放源代码的对象关系映射框架;对JDBC进行了非常轻量级的对象封装;它将POJO与数据库表建立映射关系，是一个全自动的orm框架,hibernate可以自动生成SQL语句，自动执行,跨数据库；

  2. hibernate配置文件：主配置文件： 1个

    • 连接数据库信息；hibernate运行参数,指定映射文件的位置
    • 映射文件： 若干;实体类和数据库表的映射;实体类的属性和数据库表的列 映射

  3. hibernate 访问数据库7步走:

  1. 面向对象编程中2种关系；

    • 关联:(一个类持有另一个类的对象)
    • 泛化:(继承)

  2. 多对一

    • 数据库：街道表有外键字段指向区表的主键 district_id
    • 实体类 ： 街道的实体类增加区实体类的对象属性；setter getter
      • 删除了 HOusess setter getter
      • 删除 Inteter districtId setter getter
      • 增加： private District district; setter getter
      • 区的实体类，没有任何变化；

二、spring框架

1.spring（春天 弹簧 泉水）

  1. 开源综合性质的企业级J2EE框架；是轻量级的IoC和AOP的容器框架。
  2. 备注
  • Spring 与 Struts Hibernate 等单层框架不同。Spring 致力于以一个统一的高效的方式构造整个应用，并且可以将单层框架以最佳的方式揉和在一起。建立一个连贯的体系，即提供更完善的开发环境的框架，可以为POJO对象提供企业级的服务。

  1. EJB

    A. 企业级JavaBean（Enterprise JavaBean, EJB）是一个用来构筑企业级应用的服务器端可被管理组件。

    B. EJB是一个封装有某个应用程序之业务逻辑服务器端组件。

    C. 商务软件的核心部分是它的业务逻辑。业务逻辑抽象了整个商务过程的流程，并使用计算机语言将他们实现。 …… J2EE 对于这个问题的处理方法是将业务逻辑从客户端软件中抽取出来，封装在一个组件中。这个组件运行在一个独立的服务器上，客户端软件通过网络调用组件提供的服务以实现业务逻辑，而客户端软件的功能单纯到只负责发送调用请求和显示处理结果。在J2EE 中，这个运行在一个独立的服务器上，并封装了业务逻辑的组件就是EJB（Enterprise JavaBean）组件。

2.SpringMVC框架

  1. 注解:annotation

    注解 也叫元数据。一种代码级别的说明

    它可以声明在包、类、字段、方法、局部变量、方法参数等的前面，用来对这些元素进行描述、说明。

  2. spring常用的注解:

    • @Controller: 用在Action(处理器 控制器)类上，表明该类是一个Action,它能够被自动扫描加入到容器
    • @Service:用来表明该类是一个业务逻辑类， 它能够被自动扫描加入到容器
    • @Repository(仓库)：表明该类是一个Dao, 它能够被自动扫描加入到容器
    • @Transactional:用在方法上的注解。声明该业务方法会被注入事务管理代码， 如果是查询 一般写入参数readonly
    • @Transactional(readOnly=true) 要启用该注解 spring配置文件中加入
        <tx:annotation-driven transaction-manager="transactionManager"/>

3.Spring boot框架

  1. 特点:由pivotal团队提供的全新的框架,简化新的Spring应用的初始搭建，及开发过程。无需过多关注XML的配置,即提供了一堆依赖打包，并按照使用习惯解决了依赖问题。需要构建工具的支持，如maven或gradle
  2. 特点:创建独立的Spring应用,嵌入式的tomcat,简化maven配置,自动装配Spring,没有代码产生，并且对XML也没有配置要求
  3. th标签的使用

  1. 前后端分离

    1. 背景:前后端分离已成为互联网项目开发的业界标准使用方式，通过nginx+tomcat的方式（也可以中间加一个nodejs）有效的进行解耦，并且前后端分离会为以后的大型分布式架构，微服务架构、多端化服务（多种客户端，例如浏览器、车载终端、安卓、IOS等等）打下坚实的基础。这个步骤是系统架构从猿进化承认的必经之路。
    2. 核心思想：前端HTML页面通过ajax调用后台的RESTFUL API接口并使用JSON进行数据交互。
    3. Web 服务器：一般指像Nginx、Apache这类服务器，他们一般只能解析静态资源
    4. 应用服务器：一般像指Tomcat、Jetty这类服务器可以解析静态资源也可以解析动态资源，但解析静态资源的能力没有web服务器好。

  2. 同源策略（Same_orign Policy）

    • 即在同一个浏览器中访问的网站不能访问另一个网站中的数据，除非二者具有相同的Orign 即拥有相同的协议，主机地址，以及端口号。
    • 例如：
            同一域名，不同文件夹，非跨域。
      同一域名，同一文件夹，非跨域。
      不同域名，文件路径相同，跨域。
      同一域名。不同端口，跨域。
    • 前后端分离后跨域的解决方案:在后台代码添加，Filter过滤器
    • 启动类中配置对Filter的扫描
            @ServletComponentScan("过滤器所在的包名")
    • 注解@RestController
    • 相当于@ResponseBody+@Controller 返回json数据不需要在方法前面加@ResponseBody注解了，但使用@RestConroller这个注解，就不能返回jsp.html页面，返回的只是Return 里的内容。视图解析器无法解析jsp、html页面。而我们的ajax请求正好不用返回页面，只需要返回数据。

  3. pom.xml依赖的配置主要分为:两大依赖:1.模块依赖；2.框架依赖

  4. 三个部分:1.对于项目本身模块的依赖；2:对于自己公司的封装包的依赖；3：对于第三方包的依赖。

4.Springcloud框架

  1. 微服务（Microservices）

  • 是一种架构风格，一个大型复杂软件应用由一个或多个微服务组成。系统中的各个微服务可被独立部署，各个微服务之间是松耦合的。每个微服务仅关注于完成一件任务并很好地完成该任务。在所有情况下，每个任务代表着一个小的业务能力。

  1. 组成:

    1. Eureka:

      1. Spring Cloud Netflix的Eureka 是一个服务注册与发现模块。
      2. @EnableEurekaServer 启动服务注册中心,添加的位置：Springboot的启动类上。
      3. Eureka是有界面的的访问地址：localhost:端口号
      4. eureka 的配置文件application.properties 或者 application.yml 二者可以相互转化
      5. 转化工具：http://www.toyaml.com/

    2. Feign 服务消费者

      1. 服务消费者指的是服务间的调用。

      2. 基于Netflix Feign实现的声明式服务调用客户端

      3. 默认集成了ribbon 所以Feign默认实现了负载均衡。

      4. Feign采用的是基于接口的注解 所以功能的实现类为接口形式。

      5. @EnableFeignClients 开启Feign 即开启服务消费者。

          添加的位置：Springboot的启动类上。

      6. @EnableEurekaClient或@EnableDiscoveryClient

          将一个微服务注册到服务注册中心，让注册中心能够发现，扫描到该服务。

      7. 二者的区别

          @EnableEurekaClient      只适用于Eureka作为注册中心。
  @EnableDiscoveryClient   可以是其它注册中心。

      8. @FeignClient 定义Feign客户端

      9. 注意:注意事项
        Feign 不支持GET请求传递POJO 对象 解决方法

        1.将POJO拆散成一个个单独的属性放在方法参数中

        2.把方法参数变成Map 传递

        3.添加Feign 拦截器实现（待定)

  2. Ribbon 负载均衡

    • 负载均衡是高可用网络基础架构的一个关键组成部分，有了负载均衡，我们通常可以将我们的服务部署多台，然后通过负载均衡将用户的请求分发到不同的服务器用来提高网站、应用，数据库或其它服务的的性能以及可靠性。

  3. 作用 : 解决如单点故障问题，响应速度慢问题。

    1. 注：通常情况下所有的后端服务器会保证提供相同的内容，以便用户无论哪个服务器响应，都能收到一致的内容。

  4. Spring zuul 网关服务（Spring cloud Gateway）

    1. 是整个微服务架构系统的门面一样，所有的外部客户端访问都需要经过它来进行调度和过滤。
    2. 例如帮我们实现请求的过滤功能。
    3. @EnableZuulProxy 实现启动网关。
    4. 添加网关后的请求地址
        localhost:网关端口号/消费者服务名/原本的请求
